## 1. 马斯克与奥特曼开庭对决，OpenAI 命运走向高悬

**来源**：AP News / MIT Tech Review | **发布时间**：2026-04-28 06:18 北京时间（4月27日开始陪审团遴选） | **主题**：公司战略、AI安全

埃隆·马斯克与 OpenAI CEO 山姆·奥特曼之间长达数年的法律恩怨，终于于2026年4月27日在加利福尼亚州奥克兰联邦地区法院正式开庭陪审团遴选，预计最早于4月29日（周二）进行开庭陈述。

本案核心在于 OpenAI 的"灵魂之战"：马斯克于2024年8月提起民事诉讼，指控奥特曼和联合创始人格雷格·布罗克曼背弃了2015年建立 OpenAI 时的非营利使命，将公司秘密转型为商业企业。马斯克于2015至2017年间向 OpenAI 投入约3800万美元，目前 OpenAI 估值已高达8520亿美元。马斯克并非索要个人赔偿，而是要求将不确定金额支付给 OpenAI 的慈善部门，同时要求将奥特曼逐出董事会。

案件的主审法官为 Yvonne Gonzalez Rogers，陪审团仅具咨询性质，法官握有最终裁决权。庭审中一封2023年2月的关键邮件被作为证据呈堂——奥特曼写道"你是我的英雄……但当你公开攻击OpenAI时真的让我很受伤"，马斯克则回复"我听到你了，道歉。但文明的命运悬而未决。"此案不仅涉及两位科技大亨的个人恩怨，更可能对 AI 行业权力格局产生深远影响。OpenAI 将此案定性为马斯克出于"酸葡萄心理"打压竞争对手 xAI 的手段。

## 2. OpenAI 终止与微软独家合作，模型将入驻 Amazon Bedrock

**来源**：TechCrunch | **发布时间**：2026-04-27 17:40 北京时间 | **主题**：公司战略、AI基础设施

OpenAI 与微软于2026年4月27日宣布再次重新谈判双方合作协议，正式终止历史性的独家合作关系，此举从根本上改变了 AI 行业的云服务竞争格局。

新协议的核心变化包括：其一，微软对 OpenAI 的独家 IP 授权结束，改为持有至2032年的非独家许可；其二，OpenAI 产品仍优先在 Azure 上线，但现在可以向任意云服务商的客户提供所有产品；其三，微软将停止向 OpenAI 支付营收分成，而 OpenAI 继续向微软支付分成至2030年（设有上限）；微软仍持有 OpenAI 约27%的股权。

此次协议重构的直接导火索是 OpenAI 与 Amazon 于2026年2月签署的500亿美元投资协议——Amazon 要求独家权利在 AWS Bedrock 上分发 OpenAI 的 Agent 工具 Frontier，但这与微软的原有独家条款发生冲突，甚至引发微软考虑采取法律行动。新协议的达成彻底化解了这一法律风险。Amazon CEO Andy Jassy 在 X 上庆祝，表示 OpenAI 模型将在数周内通过 AWS Bedrock 正式向客户开放。这标志着 OpenAI 从"微软独家生态"向"多云开放平台"的战略转型。

## 3. 中国叫停 Meta 20亿美元收购 Manus，创始人遭限制出境

**来源**：TechCrunch / CNBC | **发布时间**：2026-04-27 21:16 北京时间 | **主题**：融资并购、公司战略

中国国家发展和改革委员会（发改委）于2026年4月27日正式宣布叫停 Meta 对 AI 初创公司 Manus 约20亿美元的收购，要求双方完全解除收购协议，成为迄今中国在跨境 AI 收购中最具影响力的干预行动之一。

Manus 由洪晓（CEO）、纪一超（首席科学家）等人于2022年在北京成立，2025年中期迁址新加坡以寻求更大的业务自由度。公司开发通用 AI 智能体，2025年3月发布首款产品，8个月内年经常性收入（ARR）突破1亿美元，曾获 Benchmark 领投7500万美元融资。Meta 于2025年12月以约20至30亿美元宣布收购，约100名 Manus 员工已于2026年3月搬入 Meta 新加坡办公室，CEO 洪晓也已直接向 Meta 首席运营官 Javier Olivan 汇报。

## 4. GitHub Copilot 宣布6月1日起全面转向按量计费，AI Credits 取代固定套餐

**来源**：GitHub 官方博客 | **发布时间**：2026-04-28 00:03 北京时间 | **主题**：产品发布、AI基础设施

GitHub 官方于2026年4月27日发布公告，宣布自2026年6月1日起，所有 GitHub Copilot 套餐将正式迁移至按量计费模式，原有的"高级请求单元（PRUs）"将被新的 **GitHub AI Credits** 取代，计费依据为实际 Token 消耗量。

新体系下，各套餐价格保持不变但包含等值积分：Copilot Pro（$10/月含$10积分）、Copilot Pro+（$39/月含$39积分）；企业版过渡期间（6-8月）可享受额外促销积分。代码补全和 Next Edit 建议功能不消耗积分，继续免费包含于所有套餐。重要变化是：原来积分用尽后会自动降级为低成本模型继续服务，新模式下积分耗尽后将直接停止该功能，需额外购买或管理员设置超额使用权限。

GitHub 官方解释称，Copilot 已从单纯的编辑器助手演变为支持"长时间多步骤 Agentic 编程会话"的平台，计算成本大幅上升，原有按请求次数的定价模式已无法可持续支撑业务。此举与 AI 行业整体涨价趋势吻合——同期 OpenAI 将 GPT-5.2 输入 Token 价格上调至约原价的4.6倍，Anthropic 也将 Claude 企业版改为动态计费。

**原文链接**：https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/

## 5. OpenAI 开源 Symphony：将 Issue Tracker 变成 AI Agent 的控制中枢

**来源**：OpenAI 官方博客 | **发布时间**：2026-04-27 08:00 北京时间 | **主题**：产品发布、AI Agent、开源项目

OpenAI 于2026年4月27日正式开源 **Symphony**，一份面向 Codex 编排的开源规范（SPEC），旨在将 Linear 等项目管理看板变成 AI 编程 Agent 的"控制中枢"，实现每个开放 Issue 自动分配 Agent、持续运行直至完成、人类只需审查结果的理想工作流。项目在发布后迅速获得超过15,000个 GitHub Stars。

Symphony 的诞生源于 OpenAI 内部实践：团队做出激进决定，仓库中所有代码均由 Codex 生成、禁止人工编写。随着规模扩大，"上下文切换"成为瓶颈——工程师同时管理超过5个 Codex 会话后生产力明显下降，系统瓶颈不再是 AI 能力而是人类注意力。Symphony 本质上是一个与语言无关的 `SPEC.md` 文件，参考实现使用 Elixir 编写，但 Codex 已自行用 TypeScript、Go、Rust、Python 等多种语言实现它。

核心工作流是将 Linear Issue 状态机（Todo → In Progress → Human Review → Done）作为 Agent 调度驱动，支持最多10个并发 Agent、自动处理 CI 监控/Rebase/冲突，以及每个 Issue 独立的 Workspace 隔离。OpenAI 内部实测显示，部分团队前三周 PR 合并数量提升500%。OpenAI 明确表示 Symphony 是参考实现，不作为独立产品维护，鼓励社区基于规范自行构建。

## 6. 欧盟要求谷歌开放 Android AI 入口，谷歌称"不当干预"

**来源**：TechXplore（AFP通讯社） | **发布时间**：2026-04-27 北京时间 | **主题**：政策法规、公司战略

欧盟委员会于2026年4月27日正式提出措施，依据《数字市场法》（DMA）要求谷歌开放 Android 生态系统，允许竞争性 AI 助手服务获得与 Google Gemini 相同级别的系统权限，包括控制应用程序、发送邮件、订餐、分享照片等深度功能交互。

欧盟的核心关切在于，谷歌通过在 Android 设备上预装 Gemini AI 并给予其优先权限，实质上构成了对 AI 助手市场的垄断性捆绑销售，剥夺了欧盟消费者自由选择 AI 服务的权利。此次为初步结论阶段（非正式调查），尚不会直接导致罚款，但若谷歌不配合改进，欧盟可进一步认定其违规。DMA 规定最高罚款可达全球年营业额的10%。谷歌强烈反对，称此举"毫无根据"，会"不必要地推高成本，破坏用户的隐私和安全保护"。此次调查于2026年1月启动；此前2025年9月谷歌刚在另一起欧盟竞争案中被罚款29.5亿欧元。

## 7. Canonical 公布 Ubuntu AI 路线图：优先本地推理，明确"Ubuntu 不做 AI 产品"

**来源**：The Verge | **发布时间**：2026-04-28 04:47 北京时间 | **主题**：产品发布、AI基础设施

全球最流行的 Linux 发行版之一 Ubuntu 的开发商 Canonical，工程副总裁 Jon Seager 于2026年4月28日发布博文，详细阐述了2026年全年逐步在 Ubuntu Linux 中引入 AI 功能的计划，标志着主流 Linux 生态正式进入 AI 整合时代。

计划涵盖两大形式：一是在后台使用 AI 模型增强现有操作系统功能（如语音转文字/文字转语音无障碍工具）；二是面向有需求用户的全新 AI 原生功能，包括代理式 AI（Agentic AI）用于故障排查和个人自动化，以及利用 LLM 帮助新用户理解并使用 Linux 桌面生态，降低入门门槛。Canonical 认为，Linux 桌面生态系统的碎片化令新用户望而却步，引入 LLM 引导有望吸引更广泛的用户群体。

值得关注的是 Canonical 的立场和原则：优先本地推理（AI 计算尽量在本地完成以保护隐私）、保证模型透明度（明确告知用户使用何种 AI 模型），并明确表态"Ubuntu 不会成为一个 AI 产品"。此外，Seager 承诺不以员工 AI 使用频率作为绩效考核标准，而以实际交付质量为准。这一系列原则表明 Canonical 在 AI 整合上采取了相对审慎且以用户为中心的路线。

## 8. 开源 AI 编程代理 Dirac 登顶 TerminalBench，成本降低64.8%

**来源**：Hacker News / 80aj.com | **发布时间**：2026-04-27 20:35 北京时间 | **主题**：开源项目、AI Agent、开发者社区

开源 AI 编程代理 **Dirac** 在 HN 上以 289 分引发开发者社区广泛关注，项目方称其在 TerminalBench 2.0 评测中使用 Gemini-3-flash-preview 模型取得 **65.2%** 的得分，超越 Google 官方基线（47.6%）和 Junie CLI 等闭源竞品，登顶该模型榜单第一。

Dirac 专注解决 AI 编程代理领域普遍存在的"长上下文导致推理退化"问题，引入了三项核心技术：精简上下文策略（减少无关信息干扰、维持模型推理质量）、哈希锚定编辑（精准定位代码变更位置）、AST 操控（基于抽象语法树进行结构化代码操作，适合复杂重构场景）。在成本方面，Dirac 的 API 调用费用平均降低64.8%，使用的是 Gemini flash 轻量级模型而非昂贵的旗舰模型，在复杂重构任务中成本优势尤为突出。

此次 Dirac 的成功，将编程 Agent 领域的竞争焦点从"谁的分数更高"拉回到"谁更省 Token、谁工程上更可靠"的现实维度，也印证了一个趋势：精心的上下文工程（Context Engineering）配合轻量级模型，可以在成本和性能之间取得远优于暴力堆算力的平衡。项目完全开源，地址：https://github.com/dirac-run/dirac。
