## 2. Claude 主动助手功能 Orbit 遭代码泄露，接管日历/GitHub/Figma 日常简报

**来源**：凤凰网科技 / 新智元 | **发布时间**：2026-05-06 12:59 北京时间 | **主题**：产品功能 / AI助手

就在 Code with Claude 大会开幕前夕，技术情报站 testingcatalog 从 Claude Web/Mobile 客户端代码中挖掘出代号 **Orbit** 的主动式 AI 助手功能，内部功能标志位命名为 `tibro enabled`（即"orbit"反向拼写）。

Orbit 定位为**主动式日报与洞察系统**，无需用户手动 prompt，可自动整合 Gmail、Slack、GitHub、Calendar、Drive、Figma 等连接器，每天主动生成个性化工作简报。以产品经理为例，早上打开 Claude 即可看到当日 Sprint Review 会议、待审核 PR、设计师的 Figma 修改以及 CFO 邮件回复等多源信息聚合摘要，配合可操作建议一键处理。

Orbit 是 Anthropic 过去四个月系统铺垫的最终形态：Connectors（拉数据）→ Plugins（接入第三方）→ Scheduled Tasks（定时触发）→ Dispatch（任务派发）→ Projects（编排）→ Computer Use（自动操作）→ Orbit（主动简报）。其差异化在于显式支持 GitHub 和 Figma，精准瞄准开发者+设计师+产品经理群体，与 OpenAI 的 ChatGPT Pulse 和 Google 即将在 Google I/O 发布的 Gemini Proactive 形成三足鼎立之势。该功能预计优先向 Max 订阅用户开放，基础设施成本（需 7×24 小时后台运行推理）是最大挑战。

## 8. Simon Willison：Vibe Coding 与 Agentic Engineering 正在危险地趋于融合

**来源**：Simon Willison's Weblog（HN 334分，361条评论）| **发布时间**：2026-05-06 | **主题**：开发者观点 / AI编程

知名开发者 Simon Willison 在 Heavybit High Leverage 播客（第9集）后撰文，坦诚披露了自己面对 AI Agent 可靠性提升后的心态转变——他曾经认为 **vibe coding**（零审查驱动编程）和 **agentic engineering**（工程师主导的高质量 AI 辅助开发）是两种截然不同的范式，但现在发现两者的边界正在"令他不安地模糊化"。

核心论点是：当 Claude Code 可以可靠地构建 SQL 查询 API 端点时，他已经开始"不再逐行审查生产级代码"了——类比大公司里工程师不会读其他团队每一行代码，而是"用用看，出了问题再深入"。但他也指出这个类比的致命缺陷：人类团队成员有职业声誉可言，Claude Code 没有。

他将此称为**"偏差正常化"**（normalization of deviance）风险：每次 AI 恰好写对了，都在无形中增加了未来在关键时刻过度信任它的概率。文中还指出：①AI 工具导致从外观无法判断软件质量——一个半小时内生成的仓库和经历数百次提交的仓库外观相同；②整个开发流程（评审、设计、测试）都是围绕"每天几百行代码"设计的，现在可以轻松达到每天2000行，上下游流程亟需重构；③ vibe coding 不适合为他人构建软件，因为涉及他人数据和利益，是不负责任的行为。

## 10. Show HN：Tilde.run — AI Agent 事务性沙箱，支持版本化回滚（HN 118分）

**来源**：Hacker News / tilde.run | **发布时间**：2026-05-06 | **主题**：开发者工具 / AI Agent 安全

由知名开源项目 lakeFS 团队打造的 **Tilde.run** 在 HN 获得 118 分、89 条评论，是专为 AI Agent 设计的**事务性沙箱运行平台**。其核心理念借鉴数据库事务的 ACID 特性：每次 Agent 运行都是一个可回滚的原子事务——成功退出则提交变更，失败退出则一切不变，无需手动备份。

核心功能包括：①**版本化组合文件系统**——将 GitHub、AWS S3、Google Drive 统一挂载为 `~/sandbox`，所有文件从第一次提交起即被版本化，支持一键回滚任意历史状态；②**网络默认拒绝**（default-deny）——阻断云元数据服务、私有网络及未授权主机，防止数据泄露和提示注入攻击；③**Agent 优先的 RBAC 权限控制**——Agent 拥有独立作用域权限（读取/需批准/禁止），而非继承用户全部权限；④**完整审计追踪**——每次变更绑定到具体人/进程/Agent，支持时间线 diff 浏览。

Tilde.run 的出现恰逢其时——在 Cursor 删库、MCP 漏洞频发的背景下，开发者社区对 AI Agent 安全执行的需求日益迫切。它通过类 Git 的文件系统原语，将"可信赖的自主执行"从愿景变为可操作方案。目前处于私人预览阶段，支持与 Claude Code、Hugging Face 集成。
