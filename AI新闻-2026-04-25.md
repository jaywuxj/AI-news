## 2. DeepSeek V4 Preview 正式发布：1M 上下文、双模型、国产芯片适配

**来源**：MIT Technology Review + knightli.com | **发布时间**：2026-04-24 北京时间 | **主题**：LLM、开源项目、AI基础设施

中国 AI 公司 DeepSeek 于 4 月 24 日发布 **DeepSeek V4 Preview**，这是自 2025 年 1 月 R1 震撼业界以来最重要的一次更新。本次发布推出两款模型：

- **V4-Pro**：1.6 万亿总参数、490 亿激活参数，定位高性能复杂推理与 Agent 编码，在 Agentic Coding 基准中达到开源模型 SOTA；
- **V4-Flash**：2840 亿总参数、130 亿激活参数，速度快、成本低，API 价格为 V4-Pro 的 1/12（$0.14 vs $1.74 / 百万输入 token）。

最核心亮点是 **1M token 超长上下文窗口**，成为本代默认标准。与前代 V3.2 相比，V4-Pro 计算效率提升至仅需原来 27% 的算力和 10% 的内存，V4-Flash 更是仅需 10% 算力和 7% 内存，通过 Token-wise 压缩和 DeepSeek 稀疏注意力机制（DSA）实现。

战略意义上，V4 是 DeepSeek **首个针对国产芯片优化的模型**，华为确认昇腾 950 系列超节点支持 V4，DeepSeek 未向英伟达和 AMD 提供预发布访问权限。MIT Tech Review 指出，这标志着中国 AI 向摆脱英伟达依赖迈出第一步，尽管训练环节可能仍依赖英伟达。API 已即时可用，旧模型将于 **2026 年 7 月 24 日**退役。

## 7. HN 热讨：Claude 4.7 忽略 Stop Hooks，开发者工具可靠性引争议

**来源**：Hacker News | **发布时间**：2026-04-24 北京时间 | **主题**：AI工具、开发者社区

Hacker News 上一则帖子（65 分，59 评论）报告：升级到 **Claude 4.7**（claude-opus-4-7）后，Claude Code 的 **stop hook** 功能开始失效。用户 LatencyKills 设置的 stop hook 逻辑是"源文件被修改后未运行测试则阻止 Claude 停止"，但 Claude 每隔几轮后就再次忽略该规则，形成循环。

社区讨论揭示了多层技术原因：①正确触发方式应使用 `exit 2` 并将内容输出到 `stderr`（而非 `stdout`），而非 `cat` 命令；② Claude 被训练为**抵抗工具返回内容中的指令**（防 prompt injection），导致 hook 的 block 消息被语义层面忽视；③ Claude 4.7 可能修改了 hook 响应 schema，导致旧格式失效。

有自称 Claude Code 团队成员的 Thariq（用户名 `trq_`）回复，建议在发生问题的 session 中使用 `/feedback` 命令提交反馈。更深层的争议是：依赖自然语言 Markdown 文件注入确定性规则，本质上是对 LLM 非确定性系统的错误用法；harness 应有更强的强制控制层，而非将所有东西都依赖上下文和语言驱动。
